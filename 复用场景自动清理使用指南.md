# 复用场景自动清理使用指南

## 核心改进

现在 `GMWaveGradientLabelView` 和 `GMGradientNameView` 已经内置了自动清理机制，**外部使用时不需要手动调用 `resetAllStates()`**，只需要正常设置属性即可。

## 自动清理机制

### 1. GMWaveGradientLabelView

当你设置 `text` 属性时，控件会自动：
- 检测文字是否改变
- 如果改变了，自动停止旧的跑马灯
- 清理旧的文字内容
- 重新启动跑马灯（如果之前启用了）

```swift
let label = GMWaveGradientLabelView()

// ✅ 第一次设置
label.text = "用户名1"
label.enableMarquee = true

// ✅ 第二次设置（自动清理旧数据）
label.text = "用户名2"  // 自动停止旧跑马灯，清理旧文字，重新启动
```

### 2. GMGradientNameView

当你调用 `configUI` 时，控件会自动：
- 停止所有动画
- 清理旧的跑马灯状态
- 设置新的文字和配置
- 重新启动动画

```swift
let nameView = GMGradientNameView()

// ✅ 第一次配置
nameView.configUI(
    text: "用户名1",
    font: UIFont.systemFont(ofSize: 16),
    level: 3,
    defaultColors: [.white]
)

// ✅ 第二次配置（自动清理旧数据）
nameView.configUI(
    text: "用户名2",  // 自动清理
    font: UIFont.systemFont(ofSize: 16),
    level: 5,
    defaultColors: [.white]
)
```

## UITableViewCell 使用（推荐方式）

### 方式 1：直接使用（最简单）

```swift
class VIPCell: UITableViewCell {
    let nameLabel = GMGradientNameView()
    
    override init(style: UITableViewCell.CellStyle, reuseIdentifier: String?) {
        super.init(style: style, reuseIdentifier: reuseIdentifier)
        contentView.addSubview(nameLabel)
        nameLabel.frame = CGRect(x: 20, y: 10, width: 200, height: 40)
    }
    
    required init?(coder: NSCoder) {
        super.init(coder: coder)
    }
    
    // ✅ 不需要 prepareForReuse，直接在 configure 中设置即可
    func configure(user: User) {
        // 直接调用 configUI，内部会自动清理
        nameLabel.configUI(
            text: user.name,
            font: UIFont.systemFont(ofSize: 16, weight: .bold),
            sameDirectionAnimationDuration: 1.4,
            level: user.level,
            defaultColors: [.white],
            isAutoScroll: false
        )
    }
}
```

### 方式 2：使用 GMWaveGradientLabelView（更灵活）

```swift
class VIPCell: UITableViewCell {
    let nameLabel = GMWaveGradientLabelView()
    
    override init(style: UITableViewCell.CellStyle, reuseIdentifier: String?) {
        super.init(style: style, reuseIdentifier: reuseIdentifier)
        contentView.addSubview(nameLabel)
        nameLabel.frame = CGRect(x: 20, y: 10, width: 200, height: 40)
    }
    
    required init?(coder: NSCoder) {
        super.init(coder: coder)
    }
    
    // ✅ 不需要 prepareForReuse
    func configure(user: User) {
        // ✅ 直接设置属性，内部会自动清理
        nameLabel.text = user.name  // 自动清理旧数据
        nameLabel.font = UIFont.systemFont(ofSize: 16, weight: .bold)
        nameLabel.gradientColors = user.level > 0 ? 
            [.systemPink, .systemPurple, .systemBlue] : [.white]
        
        if user.level > 0 {
            nameLabel.animationDuration = 4.5
            nameLabel.marqueeSpeed = 45
            nameLabel.marqueeDelay = 1.0
            nameLabel.enableMarquee = true
            nameLabel.startAnimation()
        } else {
            nameLabel.enableMarquee = false
            nameLabel.startAnimation()
        }
    }
}
```

### 之前的问题

```swift
override func prepareForReuse() {
    super.prepareForReuse()
    nameLabel.resetAllStates()  // 必须手动调用
}

func configure(user: User) {
    nameLabel.configUI(...)
}
```

### 现在的方式

```swift
// ✅ 新方式：自动清理，不需要 prepareForReuse
func configure(user: User) {
    nameLabel.configUI(...)  // 内部自动清理
}
```

## 工作原理

### GMWaveGradientLabelView

```swift
var text: String = "" {
    didSet {
        if text != oldValue {
            // 1. 自动停止旧的跑马灯
            let wasMarqueeEnabled = enableMarquee
            if marqueeDisplayLink != nil {
                stopMarquee()
            }
            
            // 2. 重建文字和布局
            rebuildAttributedTextAndLayout()
            
            // 3. 如果之前启用了跑马灯，重新启动
            if wasMarqueeEnabled {
                DispatchQueue.main.async {
                    self.startMarquee()
                }
            }
        }
    }
}
```

### GMGradientNameView

```swift
func configUI(...) {
    // 1. 先停止所有动画
    self.stopAnimation()
    self.stopMarquee()
    
    // 2. 设置新文字（触发自动清理）
    self.text = text
    
    // 3. 配置其他属性
    self.font = font
    self.gradientColors = ...
    
    // 4. 重新启动动画
    self.enableMarquee = true
    self.startAnimation()
}
```

## 常见问题

### Q1：还需要调用 resetAllStates() 吗？

A：不需要。现在内部会自动清理，除非你有特殊需求（比如想完全重置控件状态）。

### Q2：还需要在 prepareForReuse 中做什么吗？

A：不需要。直接在 configure 方法中设置新数据即可。

### Q3：如果我想手动清理怎么办？

A：可以调用 `resetAllStates()`，但通常不需要。

### Q4：性能会受影响吗？

A：不会。自动清理只在文字改变时触发，而且是必要的操作。

## 总结

现在使用 `GMWaveGradientLabelView` 和 `GMGradientNameView` 非常简单：

1. ✅ 不需要手动调用 `resetAllStates()`
2. ✅ 不需要在 `prepareForReuse` 中清理
3. ✅ 直接设置新数据，内部自动处理复用
4. ✅ 适用于所有复用场景（UITableView、UICollectionView、JXSegmentedView）

只需要记住：**每次配置时直接调用 `configUI` 或设置 `text` 属性，内部会自动清理旧数据。**
