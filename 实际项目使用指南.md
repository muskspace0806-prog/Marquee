# 实际项目使用指南

## 问题场景

你的情况：
- 文字：`GG2668🙂🙂🙂🤪`
- 文字宽度：179.33px
- 容器宽度：203px
- 结果：179.33 < 203，所以 `needsScroll: false`，跑马灯不启动

## 解决方案

### 方案 1：降低阈值（推荐）⭐⭐⭐⭐⭐

让文字达到容器 85% 宽度就启动跑马灯：

```swift
class GMGradientNameView: GMWaveGradientLabelView {
    
    func configUI(font: UIFont, level: Int, defaultColors: [UIColor]) {
        // 清理
        stopAnimation()
        enableMarquee = false
        
        // 设置属性
        self.gradientDirection = .topRightToBottomLeft
        self.font = font
        self.gradientColors = getColors(level: level, defaultColors: defaultColors)
        
        if level > 0 {
            self.animationDuration = 2.0
            self.marqueeSpeed = 20
            self.marqueeDelay = 1.0
            self.marqueeGap = 10
            
            // ✅ 关键：降低阈值
            self.marqueeThreshold = 0.85  // 85% 就启动
            
            // ✅ 延迟启用，确保布局完成
            DispatchQueue.main.async { [weak self] in
                self?.enableMarquee = true
            }
        }
    }
}
```

**效果：**
- 179.33 > 203 * 0.85 (172.55) ✅ 会启动跑马灯

        // 1. 清理状态
        stopAnimation()

            // ✅ 关键：降低阈值，让稍短的文字也能滚动
            self.marqueeThreshold = 0.85  // 85% 就启动
            
            // ✅ 延迟启用，确保 text 和 bounds 都已正确设置
            DispatchQueue.main.async { [weak self] in
                guard let self = self else { return }

## 使用方式

### 在 UITableViewCell 中使用

```swift
class VIPCell: UITableViewCell {
    let nameLabel = GMGradientNameView()
    
    override init(style: UITableViewCell.CellStyle, reuseIdentifier: String?) {
        super.init(style: style, reuseIdentifier: reuseIdentifier)
        setupUI()
    }
    
    private func setupUI() {
        contentView.addSubview(nameLabel)
        nameLabel.frame = CGRect(x: 20, y: 10, width: 200, height: 30)
    }
    
    func configure(name: String, level: Int) {
        // 先设置文字
        nameLabel.text = name
        
        // 再配置样式
        nameLabel.configUI(
            font: UIFont.systemFont(ofSize: 20, weight: .semibold),
            level: level,
            defaultColors: [.white]
        )
    }
}

// 在 ViewController 中
func tableView(_ tableView: UITableView, cellForRowAt indexPath: IndexPath) -> UITableViewCell {
    let cell = tableView.dequeueReusableCell(withIdentifier: "VIPCell") as! VIPCell
    cell.configure(name: "GG2668🙂🙂🙂🤪", level: 1)
    return cell
}

// ✅ 兜底：在 willDisplay 中再次检查
func tableView(_ tableView: UITableView, willDisplay cell: UITableViewCell, forRowAt indexPath: IndexPath) {
    if let vipCell = cell as? VIPCell {
        vipCell.nameLabel.restartMarqueeIfNeeded()
    }
}
```

## 常见问题

### Q1: 为什么有些地方能跑马灯，有些地方不行？

A: 可能原因：
1. **文字长度不同**：有些文字长，有些文字短
2. **容器宽度不同**：不同地方的 label 宽度不一样
3. **时机问题**：有些地方布局完成了，有些还没完成

解决方法：
- 统一设置 `marqueeThreshold = 0.85`
- 在关键时机调用 `restartMarqueeIfNeeded()`

### Q2: 设置了 `marqueeThreshold` 还是不行？

A: 检查调用顺序：
```swift
// ❌ 错误：先启用跑马灯，再设置阈值
nameLabel.enableMarquee = true
nameLabel.marqueeThreshold = 0.85  // 太晚了

// ✅ 正确：先设置阈值，再启用跑马灯
nameLabel.marqueeThreshold = 0.85
nameLabel.enableMarquee = true
```

### Q3: Cell 复用时跑马灯混乱？

A: 在 `configure` 方法中先清理：
```swift
func configure(name: String, level: Int) {
    // 先清理
    nameLabel.stopAnimation()
    nameLabel.enableMarquee = false
    
    // 再配置
    nameLabel.text = name
    nameLabel.configUI(font: font, level: level, defaultColors: [.white])
}
```
