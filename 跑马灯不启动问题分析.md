## 根本原因分析

### 原因 1：layoutSubviews 没有被调用

可能的情况：
1. View 的 bounds 没有改变，所以 `setNeedsLayout()` 不会触发 `layoutSubviews`
2. View 在复用时，bounds 已经是正确的，不需要重新布局
3. `layoutSubviews` 被调用了，但 `needsRefreshLayout` 已经被重置为 `false`

### 原因 2：startMarquee 在 bounds 更新前被调用

执行顺序：
1. `configUI()` 被调用
2. `enableMarquee = true` 触发 `startMarquee()`
3. `startMarquee()` 使用当前的 `bounds` 计算（可能是旧的）
4. 后续 `layoutSubviews` 被调用，但 `forceRefreshLayout()` 没有执行

### 原因 3：文字宽度等于容器宽度

当 `textWidth == containerWidth` 时：
- `textWidth > containerWidth * threshold` 返回 `false`
- 跑马灯不会启动

解决方案：
- 降低 `marqueeThreshold` 到 0.99 或更小
- 或者修改判断条件为 `>=` 而不是 `>`

## 解决方案

### 方案 1：添加兜底机制（已实现）

在 `configUI()` 中添加异步刷新：

```swift
// ✅ 兜底机制：如果 View 已经有正确的 bounds，立即刷新
if bounds.width > 0 && bounds.height > 0 && window != nil {
    DispatchQueue.main.async { [weak self] in
        guard let self = self else { return }
        
        // 只有在 layoutSubviews 还没有处理时才刷新
        if self.needsRefreshLayout {
            self.needsRefreshLayout = false
            self.forceRefreshLayout()
        }
    }
}
```

优点：
- 确保即使 `layoutSubviews` 没有被调用，也会刷新
- 使用 `needsRefreshLayout` 标志避免重复刷新

### 方案 2：降低 marqueeThreshold

```swift
self.marqueeThreshold = 0.99  // 文字达到容器 99% 就滚动
```

这样当 `textWidth == containerWidth` 时也会滚动。

### 方案 3：修改判断条件（需要修改 WaveGradientLabel.swift）

在 `startMarquee()` 中：

```swift
// ❌ 当前
let needsScroll = textWidth > containerWidth * marqueeThreshold

// ✅ 修改为
let needsScroll = textWidth >= containerWidth * marqueeThreshold
```

这样当文字宽度等于容器宽度时也会滚动。

## 临时解决方案

如果修改后还是不行，可以使用这个临时方案：

```swift
// 在设置完 nameLabel 后，手动刷新
nameLabel.configUI(...)

// 延迟刷新（兜底）
DispatchQueue.main.asyncAfter(deadline: .now() + 0.1) { [weak nameLabel] in
    nameLabel?.restartMarqueeIfNeeded()
}
```

或者在 `willDisplay` 中：

```swift
func tableView(_ tableView: UITableView, willDisplay cell: UITableViewCell, forRowAt indexPath: IndexPath) {
    if let cell = cell as? YourCell {
        cell.nameLabel.restartMarqueeIfNeeded()
    }
}
```

## 总结

当前问题的核心是：
1. `layoutSubviews` 可能没有被调用（因为 bounds 没有改变）
2. 文字宽度等于容器宽度时不会滚动（需要降低阈值）
3. 需要兜底机制确保跑马灯启动
