# 表情符号黑色问题修复总结

## 问题描述

用户反馈：文字中包含表情符号时，表情被统一渲染成黑色，而不是显示原始的多色表情。

例如："😯这是文字🐔" 中的 😯 和 🐔 显示为黑色单色。

## 根本原因

当使用自定义字体（如 `UIFont.systemFont(ofSize:weight:)`）时，某些字体可能不支持彩色表情符号的渲染，导致表情显示为单色（黑色）。

虽然代码中已经将表情放在独立的 `emojiLabel` 中，并且没有设置 `foregroundColor`，但如果字体不支持彩色表情，iOS 仍然会将表情渲染为单色。

## 修复方案

为表情符号使用系统字体（`UIFont.systemFont(ofSize:)`），而普通文字继续使用用户指定的字体。

### 修改前

```swift
if isEmoji {
    maskAttr.append(NSAttributedString(string: str, attributes: [
        .font: font,  // ❌ 使用用户指定的字体，可能不支持彩色表情
        .foregroundColor: UIColor.clear
    ]))
    emojiAttr.append(NSAttributedString(string: str, attributes: [
        .font: font   // ❌ 使用用户指定的字体
    ]))
}
```

### 修改后

```swift
if isEmoji {
    // ✅ 表情符号：使用系统字体确保彩色显示
    let emojiFont = UIFont.systemFont(ofSize: font.pointSize)
    
    maskAttr.append(NSAttributedString(string: str, attributes: [
        .font: emojiFont,  // ✅ 使用系统字体
        .foregroundColor: UIColor.clear
    ]))
    emojiAttr.append(NSAttributedString(string: str, attributes: [
        .font: emojiFont   // ✅ 使用系统字体
    ]))
} else {
    // ✅ 普通文字：使用用户指定的字体
    maskAttr.append(NSAttributedString(string: str, attributes: [
        .font: font,
        .foregroundColor: normalColorForMask
    ]))
    emojiAttr.append(NSAttributedString(string: str, attributes: [
        .font: font,
        .foregroundColor: UIColor.clear
    ]))
}
```

## 关键点

1. **表情使用系统字体**：`UIFont.systemFont(ofSize:)` 保证支持彩色表情
2. **保持字体大小一致**：使用 `font.pointSize` 确保表情和文字大小一致
3. **不设置 foregroundColor**：表情的 `NSAttributedString` 不设置 `foregroundColor`，让系统渲染原始多色
4. **普通文字不受影响**：普通文字继续使用用户指定的字体和样式

## 测试方法

### 1. 基础测试

```swift
let label = GMWaveGradientLabelView()
label.text = "😯这是文字🐔"
label.font = UIFont.systemFont(ofSize: 30, weight: .bold)
label.gradientColors = [.systemPink, .systemPurple, .systemBlue]
label.startAnimation()
```

预期结果：
- 😯 和 🐔 显示为彩色（多色表情）
- "这是文字" 显示渐变效果

### 2. 调试测试

```swift
label.debugEmojiDetection()
```

检查输出：

确认表情使用的是系统字体（`.SFUI-Regular` 或类似）。

### 3. 对比测试

创建两个 label 对比：

```swift
// Label 1: 普通 UILabel
let normalLabel = UILabel()
normalLabel.text = "😯🐔"

// Label 2: GMWaveGradientLabelView
let gradientLabel = GMWaveGradientLabelView()
gradientLabel.text = "😯🐔"
gradientLabel.font = UIFont.systemFont(ofSize: 30, weight: .bold)
```

两者的表情应该都显示为彩色。

## 其他可能的解决方案

### 方案 2：使用 Apple Color Emoji 字体

```swift
let emojiFont = UIFont(name: "Apple Color Emoji", size: font.pointSize) ?? UIFont.systemFont(ofSize: font.pointSize)
```

这是 iOS 系统专门用于表情的字体，但 `UIFont.systemFont` 已经足够。

### 方案 3：检测字体是否支持彩色表情

```swift
func fontSupportsColorEmoji(_ font: UIFont) -> Bool {
    // 检查字体是否支持彩色表情
    // 这个方法比较复杂，不推荐
}
```

## 注意事项

1. **字体大小一致**：表情和文字使用相同的 `pointSize`，确保视觉一致
2. **不影响渐变**：表情在 `maskLabel` 中是透明的，不影响渐变效果
3. **性能影响**：使用系统字体不会有性能问题
4. **兼容性**：适用于所有 iOS 版本
