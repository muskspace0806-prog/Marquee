# 阿语渐变问题调试

## 问题描述

在阿语情况下，渐变动画循环一次后，在后半段渐变色突然扩大。

## 阿语配置

```swift
gradientDirection = .horizontalLeftToRight  // 向右 →
marqueeDirection = .leftToRight             // 向右 →
textAlignment = .right                      // 右对齐
```

## 渐变层配置

### horizontalLeftToRight

```swift
// 初始 frame
gradientLayer.frame = CGRect(x: -w, y: 0, width: w * 2, height: h)

// Transform
dx = w * phase  // phase: 0 -> 1
// 所以 gradientLayer 从 x=-w 移动到 x=0

// 视觉效果
// phase=0: gradientLayer 在 x=-w，只有右半部分可见（0 到 w）
// phase=0.5: gradientLayer 在 x=-0.5w，中间部分可见
// phase=1: gradientLayer 在 x=0，左半部分可见（0 到 w）
```

## 可能的问题

### 1. Locations 不对称

当前 locations：
```
[0, 0.125, 0.25, 0.375, 0.5, 0.625, 0.75, 0.875]
```

这意味着：
- 0-0.5：第一组渐变
- 0.5-1.0：第二组渐变（重复）

但是 gradientLayer 的可见区域是：
- phase=0: 显示 gradientLayer 的 0.5-1.0 部分（右半）
- phase=1: 显示 gradientLayer 的 0-0.5 部分（左半）

### 2. 可能的修复

问题可能出在：gradientLayer 的初始位置应该让第一组渐变（0-0.5）在可见区域。

让我检查一下：
- gradientLayer.frame = CGRect(x: -w, y: 0, width: w * 2, height: h)
- 容器可见区域：x=0 到 x=w
- gradientLayer 在 x=-w 时，可见部分是 gradientLayer 的 x=w 到 x=2w
- 对应 gradientLayer 的 0.5 到 1.0 位置（第二组渐变）✅

这是正确的！因为我们希望：
- phase=0 时显示第二组渐变
- phase=1 时显示第一组渐变
- 这样循环时从第二组过渡到第一组，视觉上是连续的

## 调试步骤

1. 运行项目，切换到阿语
2. 观察控制台输出，看 phase 值
3. 注意 phase 接近 0 或 1 时的行为
4. 检查是否有 phase 跳跃

## 预期输出

```
🎨 [Gradient] phase: 0.001, duration: 0.7, direction: horizontalLeftToRight
🎨 [Gradient] phase: 0.002, duration: 0.7, direction: horizontalLeftToRight
...
🎨 [Gradient] phase: 0.998, duration: 0.7, direction: horizontalLeftToRight
🎨 [Gradient] phase: 0.999, duration: 0.7, direction: horizontalLeftToRight
🎨 [Gradient] phase: 0.001, duration: 0.7, direction: horizontalLeftToRight  // 循环
```

如果看到 phase 突然跳跃（例如从 0.5 跳到 0.0），那就是问题所在。

## 可能的原因

1. **速度切换导致的跳跃**：
   - 从 4.5 秒切换到 0.7 秒时，phase 计算可能出错

2. **gradientLayer frame 在某些情况下被重置**：
   - 检查 layoutSubviews 是否频繁调用

3. **CATransaction 问题**：
   - 可能有隐式动画干扰
