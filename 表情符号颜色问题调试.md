# 表情符号颜色问题调试

## 问题描述

用户反馈：文字中包含表情符号时，表情被统一渲染成黑色，而不是显示原始的多色表情。

## 当前实现

### 架构设计

代码使用了双 label 架构：
1. `maskLabel` - 用于渐变 mask，普通文字显示为白色，表情显示为透明
2. `emojiLabel` - 用于显示表情，表情显示原色，普通文字显示为透明

### 代码逻辑

```swift
private func rebuildAttributedTextAndLayout() {
    for ch in full {
        let isEmoji = characterContainsEmoji(ch)
        
        if isEmoji {
            // maskLabel: 表情透明
            maskAttr.append(NSAttributedString(string: str, attributes: [
                .font: font,
                .foregroundColor: UIColor.clear
            ]))
            // emojiLabel: 表情不设置颜色（应该显示原色）
            emojiAttr.append(NSAttributedString(string: str, attributes: [
                .font: font
            ]))
        } else {
            // maskLabel: 文字白色
            maskAttr.append(NSAttributedString(string: str, attributes: [
                .font: font,
                .foregroundColor: UIColor.white
            ]))
            // emojiLabel: 文字透明
            emojiAttr.append(NSAttributedString(string: str, attributes: [
                .font: font,
                .foregroundColor: UIColor.clear
            ]))
        }
    }
}
```

## 问题分析

### 可能的原因

1. **UILabel 默认 textColor**
   - 当 `NSAttributedString` 没有设置 `foregroundColor` 时，UILabel 会使用自己的 `textColor` 属性
   - UILabel 的默认 `textColor` 是黑色
   - 因此表情符号会显示为黑色

2. **iOS 版本差异**
   - 不同 iOS 版本对表情符号的渲染可能有差异
   - 某些版本可能会强制应用 `textColor`

3. **字体问题**
   - 如果字体不支持彩色表情，可能会显示为单色

## 解决方案

### 方案 1：明确设置 emojiLabel 的 textColor（当前尝试）

```swift
// 在 setupLayers() 中
emojiLabel.textColor = .black  // 设置默认颜色
emojiLabel2.textColor = .black
```

但这可能不够，因为黑色仍然会覆盖表情的原色。

### 方案 2：使用 NSAttributedString 的特殊属性

iOS 中，要让表情显示原色，需要确保：
1. 不设置 `foregroundColor` 属性
2. UILabel 的 `textColor` 不影响表情

但实际上，UILabel 的 `textColor` 总是会影响没有设置 `foregroundColor` 的文字。

### 方案 3：使用 UIColor.label 或系统颜色

```swift
emojiLabel.textColor = .label  // 使用系统默认颜色
```

这样可以适应深色/浅色模式，但仍然可能不显示多色表情。

### 方案 4：检查表情符号检测逻辑

可能某些表情没有被正确识别为表情，导致被当作普通文字处理。

```swift
private func isEmojiScalar(_ scalar: UnicodeScalar) -> Bool {
    if scalar.properties.isEmojiPresentation { return true }
    if scalar.properties.generalCategory == .otherSymbol, scalar.properties.isEmoji { return true }
    if scalar.properties.isEmoji && (scalar.value >= 0x1F000) { return true }
    return false
}
```

### 方案 5：完全不设置 emojiLabel 的 textColor

让 UILabel 使用默认值，但在 `NSAttributedString` 中也不设置任何颜色。

## 可能的根本原因

在 iOS 中，表情符号（Emoji）是特殊的 Unicode 字符，它们有自己的颜色信息。但是：

1. **如果设置了 foregroundColor**：表情会被渲染为单色（使用指定的颜色）
2. **如果没有设置 foregroundColor**：
   - 如果 UILabel 有 textColor，表情会使用这个颜色（单色）
   - 如果 UILabel 没有 textColor（或使用默认值），表情应该显示多色

问题可能是：我们的 `emojiLabel` 继承了某个默认的 `textColor`，导致表情被渲染为单色。
