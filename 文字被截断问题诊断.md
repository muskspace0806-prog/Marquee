# 文字被截断问题诊断

## 问题描述

你遇到的情况：
- 文字：`123456789`
- 期望：完整显示或滚动显示完整文字
- 实际：显示成 `12345...`（被截断）

## 可能的原因

### 1. 容器宽度设置错误 ⭐⭐⭐⭐⭐

**最常见的原因！**

```swift
// 调试输出
label.debugMarqueeStatus()

// 检查输出：
// - frame: (x, y, width, height)  // 实际的 frame
// - bounds: (0, 0, width, height) // bounds
// - textWidth: XXXpx              // 文字实际需要的宽度
```

**可能的情况：**

#### 情况 A：frame 设置太小
```swift
// ❌ 错误：frame 宽度太小
label.frame = CGRect(x: 20, y: 100, width: 100, height: 30)  // 只有 100px
label.text = "123456789"  // 需要 150px

// 结果：显示 "12345..."
```

**解决方法：**
```swift
// 方法1：增大 frame
label.frame = CGRect(x: 20, y: 100, width: 200, height: 30)

// 方法2：根据文字自动计算宽度
let textWidth = (label.text as NSString).size(withAttributes: [.font: label.font]).width
label.frame = CGRect(x: 20, y: 100, width: textWidth + 20, height: 30)
```

#### 情况 B：Auto Layout 约束太小
```swift
// ❌ 错误：约束限制了宽度
label.widthAnchor.constraint(equalToConstant: 100).isActive = true

// 结果：即使文字很长，也只显示 100px
```

**解决方法：**
```swift
// 方法1：增大约束
label.widthAnchor.constraint(equalToConstant: 200).isActive = true

// 方法2：使用不等式约束
label.widthAnchor.constraint(lessThanOrEqualToConstant: 300).isActive = true

// 方法3：让宽度自适应
// 不设置 width 约束，只设置 leading 和 trailing
label.leadingAnchor.constraint(equalTo: view.leadingAnchor, constant: 20).isActive = true
label.trailingAnchor.constraint(equalTo: view.trailingAnchor, constant: -20).isActive = true
```

### 2. numberOfLines 设置问题 ⭐⭐⭐

```swift
// 检查 numberOfLines
print("numberOfLines: \(label.numberOfLines)")
```

WaveGradientLabelView 内部的 maskLabel 和 emojiLabel 都设置了 `numberOfLines = 1`，这是正确的。

但如果你在外部修改了，可能会有问题：
```swift
// ❌ 不要这样做
label.numberOfLines = 0  // 多行模式
```

### 3. lineBreakMode 问题 ⭐⭐

```swift
// 检查 lineBreakMode
print("lineBreakMode: \(label.maskLabel.lineBreakMode)")
```

如果设置成了 `.byTruncatingTail`，会显示 "12345..."

**解决方法：**
```swift
// 在 WaveGradientLabelView 中添加
maskLabel.lineBreakMode = .byClipping  // 或 .byWordWrapping
```

### 4. clipsToBounds 问题 ⭐

```swift
// 检查 clipsToBounds
print("clipsToBounds: \(label.clipsToBounds)")
```

WaveGradientLabelView 默认设置了 `clipsToBounds = true`，这是正确的。

但如果你在外部修改了：
```swift
// ❌ 不要这样做
label.clipsToBounds = false
```

### 5. 父视图裁剪问题 ⭐⭐

```swift
// 检查父视图
print("superview.clipsToBounds: \(label.superview?.clipsToBounds ?? false)")
print("superview.bounds: \(label.superview?.bounds ?? .zero)")
```

如果父视图的 `clipsToBounds = true` 且宽度太小，会裁剪子视图。

## 诊断步骤

### 第一步：打印详细信息

```swift
label.debugMarqueeStatus()
```

**重点关注：**
```
- frame: (20.0, 100.0, 100.0, 30.0)  // ⚠️ 宽度只有 100
- textWidth: 150.0px                  // ⚠️ 文字需要 150
- containerWidth: 100.0px             // ⚠️ 容器只有 100
```

如果 `textWidth > containerWidth`，说明容器太小了！

### 第二步：检查是否使用 Auto Layout

```swift
print("Using Auto Layout: \(label.translatesAutoresizingMaskIntoConstraints == false)")
```

如果是 `true`，检查约束：
```swift
// 打印所有约束
print("Constraints: \(label.constraints)")
```

### 第三步：检查父视图

```swift
print("Superview: \(label.superview)")
print("Superview bounds: \(label.superview?.bounds ?? .zero)")
```

## 解决方案

### 方案 1：不使用 marqueeThreshold（推荐）⭐⭐⭐⭐⭐

**如果文字确实比容器宽，不应该使用 marqueeThreshold！**

```swift
func configUI(font: UIFont, level: Int, defaultColors: [UIColor]) {
    // ... 其他代码 ...
    
    if level > 0 {
        // ❌ 不要设置 marqueeThreshold
        // self.marqueeThreshold = 0.85
        
        // ✅ 使用默认值 1.0
        self.marqueeThreshold = 1.0  // 或者不设置，默认就是 1.0
        
        DispatchQueue.main.async { [weak self] in
            self?.enableMarquee = true
        }
    }
}
```

### 方案 2：增大容器宽度

```swift
// 方法1：直接设置 frame
label.frame = CGRect(x: 20, y: 100, width: 250, height: 30)  // 增大宽度

// 方法2：Auto Layout
label.widthAnchor.constraint(equalToConstant: 250).isActive = true

// 方法3：自适应宽度
label.leadingAnchor.constraint(equalTo: view.leadingAnchor, constant: 20).isActive = true
label.trailingAnchor.constraint(equalTo: view.trailingAnchor, constant: -20).isActive = true
```

### 方案 3：根据文字动态设置宽度

```swift
func configure(name: String, level: Int) {
    nameLabel.text = name
    
    // 计算文字宽度
    let textWidth = (name as NSString).size(withAttributes: [
        .font: UIFont.systemFont(ofSize: 20, weight: .semibold)
    ]).width
    
    // 设置 frame（留一些边距）
    nameLabel.frame = CGRect(x: 20, y: 100, width: min(textWidth + 20, 300), height: 30)
    
    // 配置样式
    nameLabel.configUI(font: UIFont.systemFont(ofSize: 20, weight: .semibold),
                       level: level,
                       defaultColors: [.white])
}
```

### 方案 4：检查并修复 lineBreakMode

在 WaveGradientLabelView 中添加：

```swift
private func setupLayers() {
    // ... 现有代码 ...
    
    // ✅ 确保不会显示省略号
    maskLabel.lineBreakMode = .byClipping
    maskLabel2.lineBreakMode = .byClipping
    emojiLabel.lineBreakMode = .byClipping
    emojiLabel2.lineBreakMode = .byClipping
}
```

## 实际案例分析

### 案例 1：容器太小

```
🔍 [Marquee Debug]
   - text: "123456789"
   - frame: (20.0, 100.0, 100.0, 30.0)  // ⚠️ 宽度 100
   - textWidth: 150.0px                  // ⚠️ 需要 150
   - containerWidth: 100.0px
   - needsScroll: false                  // ⚠️ 因为 marqueeThreshold = 0.85
```

**问题：**
- 文字需要 150px
- 容器只有 100px
- 因为设置了 `marqueeThreshold = 0.85`，判断为不需要滚动
- 结果：文字被截断成 "12345..."

**解决方法：**
```swift
// 方法1：移除 marqueeThreshold
label.marqueeThreshold = 1.0

// 方法2：增大容器
label.frame = CGRect(x: 20, y: 100, width: 200, height: 30)
```

### 案例 2：Auto Layout 约束冲突

```
🔍 [Marquee Debug]
   - frame: (20.0, 100.0, 150.0, 30.0)
   - textWidth: 200.0px
   - containerWidth: 150.0px
   ⚠️ Using Auto Layout - constraints may not be applied yet
```

**问题：**
- 使用了 Auto Layout
- 约束限制了宽度为 150px
- 文字需要 200px

**解决方法：**
```swift
// 检查约束
label.constraints.forEach { print($0) }

// 修改约束
label.widthAnchor.constraint(equalToConstant: 250).isActive = true
```

## 推荐配置

### 对于你的场景（VIP 名字）

```swift
class GMGradientNameView: GMWaveGradientLabelView {
    
    func configUI(font: UIFont, level: Int, defaultColors: [UIColor]) {
        stopAnimation()
        enableMarquee = false
        
        self.gradientDirection = .topRightToBottomLeft
        self.font = font
        self.gradientColors = getColors(level: level, defaultColors: defaultColors)
        
        if level > 0 {
            self.animationDuration = 2.0
            self.marqueeSpeed = 20
            self.marqueeDelay = 1.0
            self.marqueeGap = 10
            
            // ✅ 不设置 marqueeThreshold，使用默认值 1.0
            // self.marqueeThreshold = 1.0  // 可以不写，默认就是 1.0
            
            DispatchQueue.main.async { [weak self] in
                self?.enableMarquee = true
                
                #if DEBUG
                self?.debugMarqueeStatus()
                #endif
            }
        }
    }
}
```

### 使用时确保容器足够大

```swift
// 方法1：固定宽度（确保足够大）
nameLabel.frame = CGRect(x: 20, y: 100, width: 250, height: 30)

// 方法2：动态计算
let maxWidth: CGFloat = 300
let textWidth = (name as NSString).size(withAttributes: [.font: font]).width
let labelWidth = min(textWidth + 20, maxWidth)
nameLabel.frame = CGRect(x: 20, y: 100, width: labelWidth, height: 30)

// 方法3：Auto Layout（不限制宽度）
nameLabel.leadingAnchor.constraint(equalTo: view.leadingAnchor, constant: 20).isActive = true
nameLabel.trailingAnchor.constraint(lessThanOrEqualTo: view.trailingAnchor, constant: -20).isActive = true
```

## 总结

**你的问题不是 `marqueeThreshold`，而是容器宽度设置有问题！**

1. ✅ 移除 `marqueeThreshold = 0.85`，使用默认值 1.0
2. ✅ 确保容器宽度足够大（至少和文字一样宽）
3. ✅ 调用 `debugMarqueeStatus()` 查看详细信息
4. ✅ 检查 frame、bounds、约束是否正确

如果文字确实比容器宽，跑马灯会自动启动，不需要调整阈值！
