# 跑马灯问题排查指南

## 症状：渐变动画正常，但跑马灯不工作

如果渐变动画能正常运行，说明 view 的基本状态是正常的（已添加到 window、bounds 正确等）。跑马灯不工作通常是以下几个原因：

## 常见原因排查

### 1. 文字太短，不需要滚动 ⭐⭐⭐⭐⭐

**最常见的原因！**

跑马灯只在文字宽度 > 容器宽度时才会启动。

**排查方法：**
```swift
// 在设置跑马灯后立即调用
label.debugMarqueeStatus()
```

**输出示例：**
```
🔍 [Marquee Debug]
   - enableMarquee: true
   - text: "短文本"
   - maskLabel.frame: (0.0, 0.0, 120.0, 60.0)  // 文字宽度
   - bounds: (0.0, 0.0, 300.0, 60.0)           // 容器宽度
   - needsScroll: false  ⚠️ 文字太短！
```

**解决方法：**
- 使用更长的文字
- 减小容器宽度
- 增大字体大小

### 2. maskLabel 的 attributedText 为空或计算错误 ⭐⭐⭐⭐

**原因：**
- `text` 为空字符串
- `font` 未正确设置
- `rebuildAttributedTextAndLayout()` 未被调用

**排查方法：**
```swift
label.debugMarqueeStatus()
// 检查输出中的：
// - text: 是否为空
// - maskLabel.attributedText: 是否为 nil
// - maskLabel.frame.width: 是否为 0
```

**解决方法：**
```swift
// 确保按正确顺序设置
label.font = UIFont.systemFont(ofSize: 35, weight: .heavy)  // 先设置字体
label.text = "这是一段很长的文字"  // 再设置文字
label.enableMarquee = true  // 最后启用跑马灯
```

### 3. bounds.height 为 0 ⭐⭐⭐

**原因：**
- Auto Layout 约束未生效
- frame 未正确设置
- 在 `layoutSubviews` 之前就启用了跑马灯

**排查方法：**
```swift
label.debugMarqueeStatus()
// 检查 bounds: (0.0, 0.0, 300.0, 0.0)  ⚠️ height 为 0！
```

**解决方法：**
```swift
// 方法1：确保 frame 正确
label.frame = CGRect(x: 20, y: 100, width: 300, height: 60)

// 方法2：Auto Layout 后强制布局
view.layoutIfNeeded()
label.enableMarquee = true

// 方法3：延迟启用
DispatchQueue.main.async {
    label.enableMarquee = true
}
```

### 4. CADisplayLink 的 RunLoop Mode 问题 ⭐⭐

**原因：**
- 在某些特殊场景下（如 UIScrollView 滚动时），`.common` mode 可能不够

**排查方法：**
```swift
// 检查是否在滚动视图中
if let scrollView = label.superview as? UIScrollView {
    print("⚠️ Label is in a scroll view")
}
```

**当前代码已使用 `.common` mode，通常不会有问题。**

### 5. maskLabel 和 emojiLabel 的层级问题 ⭐⭐

**原因：**
- maskLabel 是 CALayer，不在 view 层级中
- emojiLabel 是 UILabel，在 view 层级中
- 如果 emojiLabel 的 frame 不正确，可能看不到滚动效果

**排查方法：**
```swift
label.debugMarqueeStatus()
// 检查：
// - maskLabel.frame: 应该有正确的宽度
// - emojiLabel.frame: 应该和 maskLabel.frame 一致
```

### 6. 字体加载问题 ⭐

**原因：**
- 使用自定义字体但字体文件未正确加载
- 字体名称错误

**排查方法：**
```swift
print("Font: \(label.font)")
print("Font family: \(label.font.familyName)")
print("Font name: \(label.font.fontName)")

// 检查字体是否真的加载了
let testSize = ("测试" as NSString).size(withAttributes: [.font: label.font])
print("Test size: \(testSize)")  // 如果为 0，说明字体有问题
```

### 7. clipsToBounds 或 masksToBounds 被修改 ⭐

**原因：**
- 外部代码修改了 `clipsToBounds`
- 跑马灯需要 `clipsToBounds = true` 来裁剪超出部分

**排查方法：**
```swift
print("clipsToBounds: \(label.clipsToBounds)")  // 应该是 true
```

**解决方法：**
```swift
label.clipsToBounds = true
```

## 调试步骤

### 第一步：使用调试方法

```swift
// 在怀疑有问题的地方调用
label.debugMarqueeStatus()
```

### 第二步：检查输出

重点关注以下几项：

1. **needsScroll**: 如果是 `false`，说明文字太短
2. **maskLabel.frame.width**: 如果是 0，说明文字或字体有问题
3. **bounds**: 如果 width 或 height 是 0，说明布局有问题
4. **window**: 如果是 `❌ not attached`，说明 view 还没添加到 window
5. **marqueeDisplayLink**: 如果是 `❌ not running`，说明跑马灯没启动

### 第三步：根据输出定位问题

```
⚠️ WARNING: Marquee enabled but not running!
```

如果看到这个警告，说明跑马灯应该运行但没有运行，需要进一步排查。

## 实际案例

### 案例 1：文字太短

```swift
// ❌ 错误：文字太短
label.text = "短文本"
label.enableMarquee = true
// 跑马灯不会启动，因为不需要滚动

// ✅ 正确：使用足够长的文字
label.text = "这是一段很长的文字，需要滚动才能看完"
label.enableMarquee = true
```

### 案例 2：布局时机问题

```swift
// ❌ 错误：bounds 还是 zero
let label = WaveGradientLabelView()
label.text = "很长的文字"
label.enableMarquee = true  // 此时 bounds 可能还是 zero
view.addSubview(label)

// ✅ 正确：先设置 frame 或等待布局完成
let label = WaveGradientLabelView(frame: CGRect(x: 20, y: 100, width: 300, height: 60))
label.text = "很长的文字"
label.enableMarquee = true
view.addSubview(label)
```

### 案例 3：Auto Layout

```swift
// ❌ 错误：约束还没生效
let label = WaveGradientLabelView()
label.text = "很长的文字"
view.addSubview(label)

// 设置约束
label.translatesAutoresizingMaskIntoConstraints = false
NSLayoutConstraint.activate([...])

label.enableMarquee = true  // 约束可能还没生效

// ✅ 正确：强制布局后再启用
view.layoutIfNeeded()
label.enableMarquee = true

// 或者延迟启用
DispatchQueue.main.async {
    label.enableMarquee = true
}
```

### 案例 4：Cell 复用

```swift
// ❌ 错误：没有重置状态
func configure(with text: String) {
    label.text = text
    // 如果之前的 cell 没有启用跑马灯，这个 cell 也不会启用
}

// ✅ 正确：每次都重新设置
func configure(with text: String) {
    label.text = text
    label.enableMarquee = true
    label.restartMarqueeIfNeeded()
}
```

### 案例 5：异步加载

```swift
// ❌ 错误：文字还没加载完就启用跑马灯
label.enableMarquee = true
fetchText { text in
    label.text = text  // 文字更新了，但跑马灯已经判断过不需要滚动
}

// ✅ 正确：文字加载完后再启用
fetchText { text in
    label.text = text
    label.enableMarquee = true
    DispatchQueue.main.async {
        label.restartMarqueeIfNeeded()
    }
}
```

## 快速检查清单

在实际项目中遇到跑马灯不工作时，按以下顺序检查：

- [ ] 1. 调用 `label.debugMarqueeStatus()` 查看状态
- [ ] 2. 检查文字是否足够长（maskLabel.width > bounds.width）
- [ ] 3. 检查 bounds 是否正确（width > 0, height > 0）
- [ ] 4. 检查 view 是否已添加到 window
- [ ] 5. 检查 font 是否正确设置
- [ ] 6. 检查 text 是否为空
- [ ] 7. 尝试调用 `label.restartMarqueeIfNeeded()`
- [ ] 8. 检查是否在正确的时机启用跑马灯（布局完成后）

## 终极解决方案

如果以上都检查过了还是不行，使用这个万能方法：

```swift
// 延迟启用，确保所有布局都完成
DispatchQueue.main.asyncAfter(deadline: .now() + 0.1) {
    label.enableMarquee = false
    label.enableMarquee = true
    label.restartMarqueeIfNeeded()
    
    // 再次检查状态
    label.debugMarqueeStatus()
}
```
