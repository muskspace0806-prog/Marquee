# 文字省略号显示说明

## 问题描述

当 `enableMarquee = false` 时，如果文字超出控件宽度，之前会被直接截断，没有显示省略号（...），导致用户不知道文字被截断了。

## 解决方案

现在控件会根据 `enableMarquee` 的状态自动切换 `lineBreakMode`：

- **启用跑马灯时**（`enableMarquee = true`）：使用 `.byClipping`
  - 不显示省略号
  - 因为文字会滚动显示完整内容
  
- **禁用跑马灯时**（`enableMarquee = false`）：使用 `.byTruncatingTail`
  - 显示省略号（...）
  - 让用户知道文字被截断了

## 使用示例

### 示例 1：禁用跑马灯（显示省略号）

```swift
let label = GMWaveGradientLabelView()
label.frame = CGRect(x: 20, y: 100, width: 150, height: 40)
label.text = "这是一段很长的文字内容"
label.font = UIFont.systemFont(ofSize: 16)
label.gradientColors = [.systemPink, .systemPurple]
label.enableMarquee = false  // ✅ 禁用跑马灯
label.startAnimation()

// 结果：显示 "这是一段很长的文字..."
```

### 示例 2：启用跑马灯（不显示省略号）

```swift
let label = GMWaveGradientLabelView()
label.frame = CGRect(x: 20, y: 100, width: 150, height: 40)
label.text = "这是一段很长的文字内容"
label.font = UIFont.systemFont(ofSize: 16)

label.enableMarquee = true  // ✅ 启用跑马灯
label.marqueeSpeed = 45
label.startAnimation()

// 结果：文字会滚动显示完整内容，不显示省略号
```

## 技术实现

### 核心代码

```swift
var enableMarquee: Bool = false {
    didSet {
        // ✅ 根据跑马灯状态设置 lineBreakMode
        updateLineBreakMode()
        
        if enableMarquee {
            startMarquee()
        } else {
            stopMarquee()
        }
    }
}

private func updateLineBreakMode() {
    // 启用跑马灯时：不截断（.byClipping），因为会滚动显示完整文字
    // 禁用跑马灯时：显示省略号（.byTruncatingTail），避免文字超出
    let mode: NSLineBreakMode = enableMarquee ? .byClipping : .byTruncatingTail
    maskLabel.lineBreakMode = mode
    maskLabel2.lineBreakMode = mode
    emojiLabel.lineBreakMode = mode
    emojiLabel2.lineBreakMode = mode
}
```

### 初始化时的默认值

```swift
private func setupLayers() {
    // ...
    
    // ✅ 默认使用 .byTruncatingTail（显示省略号）
    maskLabel.lineBreakMode = .byTruncatingTail
    maskLabel2.lineBreakMode = .byTruncatingTail
    emojiLabel.lineBreakMode = .byTruncatingTail
    emojiLabel2.lineBreakMode = .byTruncatingTail
    
    // ...
}
```

## 注意事项

1. **自动切换**：`lineBreakMode` 会根据 `enableMarquee` 自动切换，不需要手动设置

2. **省略号位置**：使用 `.byTruncatingTail`，省略号显示在文字末尾

3. **其他截断模式**：如果需要其他截断模式（如 `.byTruncatingHead` 或 `.byTruncatingMiddle`），可以修改 `updateLineBreakMode()` 方法

4. **表情符号**：省略号对表情符号也有效，不会截断表情符号的中间
