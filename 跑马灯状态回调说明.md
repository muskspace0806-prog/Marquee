# 跑马灯状态回调说明

## 功能说明

`marqueeStatusChanged` 是一个回调 block，当跑马灯状态改变时（启动/停止）会自动调用。

## 使用方法

### 基础用法

```swift
let nameLabel = GMWaveGradientLabelView()
nameLabel.text = "用户名"

// ✅ 设置回调
nameLabel.marqueeStatusChanged = { isScrolling in
    if isScrolling {
        print("跑马灯已启动")
    } else {
        print("跑马灯已停止")
    }
}

// 配置跑马灯
nameLabel.enableMarquee = true
```

### 在 UITableViewCell 中使用

```swift
class VIPCell: UITableViewCell {
    let nameLabel = GMWaveGradientLabelView()
    
    func configure(name: String, level: Int) {
        nameLabel.text = name
        
        // ✅ 设置回调
        nameLabel.marqueeStatusChanged = { [weak self] isScrolling in
            guard let self = self else { return }
            
            if isScrolling {
                // 跑马灯启动时的处理
                print("Cell 的跑马灯已启动")
            } else {
                // 跑马灯停止时的处理
                print("Cell 的跑马灯已停止")
            }
        }
        
        nameLabel.configUI(font: font, level: level, defaultColors: [.white])
    }
}
```

### 在自定义类中使用

```swift
class GMGradientNameView: WaveGradientLabelView {
    
    var onMarqueeStatusChanged: ((Bool) -> Void)?
    
    func configUI(font: UIFont, level: Int, defaultColors: [UIColor]) {
        // 配置样式
        self.gradientDirection = .horizontalRightToLeft
        self.font = font
        self.gradientColors = getColors(level: level, defaultColors: defaultColors)
        
        if level > 0 {
            self.animationDuration = 4.5
            self.sameDirectionAnimationDuration = 0.7
            self.marqueeSpeed = 45
            self.marqueeDelay = 1.0
            
            // ✅ 设置回调，转发给外部
            self.marqueeStatusChanged = { [weak self] isScrolling in
                self?.onMarqueeStatusChanged?(isScrolling)
            }
            
            self.enableMarquee = true
        }
    }
}

// 使用
let nameLabel = GMGradientNameView()
nameLabel.onMarqueeStatusChanged = { isScrolling in
    print("跑马灯状态：\(isScrolling ? "滚动中" : "已停止")")
}
```

## 回调时机

### 启动时（isScrolling = true）

回调会在以下情况触发：
1. `enableMarquee = true` 且文字超出容器宽度
2. 调用 `restartMarqueeIfNeeded()` 且成功启动
3. 调用 `forceEnableMarqueeIfTextOverflow()` 且成功启动

### 停止时（isScrolling = false）

回调会在以下情况触发：
1. `enableMarquee = false`
2. 文字变短，不再需要滚动
3. 组件被移除或隐藏

## 使用场景

### 场景 1：统计跑马灯使用情况

```swift
nameLabel.marqueeStatusChanged = { isScrolling in
    if isScrolling {
        // 上报埋点：跑马灯启动
        Analytics.track("marquee_started", properties: [
            "text_length": nameLabel.text.count,
            "user_level": userLevel
        ])
    }
}
```

### 场景 2：根据跑马灯状态调整 UI

```swift
nameLabel.marqueeStatusChanged = { [weak self] isScrolling in
    guard let self = self else { return }
    
    if isScrolling {
        // 跑马灯启动时，隐藏其他元素
        self.iconView.isHidden = true
    } else {
        // 跑马灯停止时，显示其他元素
        self.iconView.isHidden = false
    }
}
```

### 场景 3：调试和日志

```swift

```

### 场景 4：性能监控

```swift
var marqueeStartTime: Date?

```

## View 复用场景

如果你的 View 会被复用（例如在 UITableViewCell 或自定义 View 中），需要在重新配置前调用 `resetAllStates()`：

```swift
class VIPCell: UITableViewCell {
    let nameLabel = GMWaveGradientLabelView()
    
    override func prepareForReuse() {
        super.prepareForReuse()
        // ✅ 重置所有状态，避免状态混乱
        nameLabel.resetAllStates()
    }
    
    func configure(name: String, level: Int) {
        // ✅ 重新设置回调
        nameLabel.marqueeStatusChanged = { [weak self] isScrolling in
            print("状态改变: \(isScrolling)")
        }
        
        // 配置新内容
        nameLabel.text = name
        nameLabel.configUI(font: font, level: level, defaultColors: [.white])
    }
}
```

### 为什么需要 resetAllStates()？

当多个 View 继承自同一个基类并复用时，可能会出现：
- 文字内容不匹配（`debugMarqueeStatus()` 显示 TEXT MISMATCH）
- 跑马灯状态混乱（应该滚动但没有滚动）
- 回调触发异常

`resetAllStates()` 会清理：
- 停止所有动画
- 重置文字内容
- 清空回调
- 隐藏第二份内容
- 重置 frame

## 注意事项

1. **弱引用**：在闭包中使用 `[weak self]` 避免循环引用
   ```swift
   nameLabel.marqueeStatusChanged = { [weak self] isScrolling in
       guard let self = self else { return }
       // 使用 self
   }
   ```

2. **线程安全**：回调在主线程调用，可以直接更新 UI

3. **多次调用**：如果跑马灯重启，会先调用 `false`，再调用 `true`

4. **文字不需要滚动**：如果文字宽度小于容器宽度，不会触发回调

5. **View 复用**：在复用场景中，先调用 `resetAllStates()`，再重新设置回调和配置

## 完整示例

```swift
class ProfileViewController: UIViewController {
    let nameLabel = GMWaveGradientLabelView()
    
    override func viewDidLoad() {
        super.viewDidLoad()
        
        // 添加到视图
        view.addSubview(nameLabel)
        nameLabel.frame = CGRect(x: 20, y: 100, width: 200, height: 40)
        
        // 设置回调
        nameLabel.marqueeStatusChanged = { [weak self] isScrolling in
            self?.handleMarqueeStatusChanged(isScrolling)
        }
        
        // 配置
        nameLabel.text = "VIP用户名很长需要滚动显示"
        nameLabel.gradientColors = [.systemPink, .systemPurple, .systemBlue]
        nameLabel.enableMarquee = true
        nameLabel.startAnimation()
    }
    
    private func handleMarqueeStatusChanged(_ isScrolling: Bool) {
        if isScrolling {
            print("✅ 跑马灯已启动")
            // 可以在这里做一些处理，比如：
            // - 上报埋点
            // - 调整 UI
            // - 记录日志
        } else {
            print("⏹ 跑马灯已停止")
        }
    }
}
```
