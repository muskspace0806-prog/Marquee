# 复用场景跑马灯不启动问题修复

## 问题现象

在 JXSegmentedView、UITableView、UICollectionView 等复用场景中：
- 文字显示正确
- 但跑马灯没有启动（`enableMarquee: false`）
- 调试输出显示 `marqueeDisplayLink: ❌ not running`

## 根本原因

### 原因 1：bounds 时序问题

```swift
// ❌ 错误的顺序
nameLabel.text = "用户名"
nameLabel.configUI(...)  // 此时 bounds 可能还是旧的
// 异步调用 forceRefreshLayout()
DispatchQueue.main.async {
    nameLabel.forceRefreshLayout()  // bounds 已更新，但 enableMarquee 可能被重置
}
```

问题：
1. `configUI()` 被调用时，`bounds` 可能还是复用前的旧值
2. `updateTextSize()` 使用旧的 `bounds` 计算文字宽度
3. 异步调用 `forceRefreshLayout()` 时，`enableMarquee` 可能已经被其他代码重置为 `false`

### 原因 2：enableMarquee 被意外重置

从调试输出可以看到：
```
- enableMarquee before refresh: false  // ❌ 应该是 true
- enableMarquee after refresh: false   // ❌ 还是 false
```

可能的原因：
1. `resetAllStates()` 被多次调用
2. 某个属性的 `didSet` 触发了 `enableMarquee = false`
3. 异步调用之间有其他代码修改了 `enableMarquee`

## 解决方案

### 方案 1：使用 layoutSubviews 刷新（推荐）

```swift
class GMGradientNameView: GMWaveGradientLabelView {
    private var needsRefreshLayout = false
    
    func configUI(...) {
        // 1. 重置状态
        self.resetAllStates()
        
        // 2. 配置属性
        self.font = font
        self.gradientColors = colors
        self.enableMarquee = true
        
        // 3. 标记需要刷新
        needsRefreshLayout = true
    }
    
    override func layoutSubviews() {
        super.layoutSubviews()
        
        // 4. 在布局完成后刷新
        if needsRefreshLayout {
            needsRefreshLayout = false
            forceRefreshLayout()
        }
    }
}
```

优点：
- ✅ 确保在 `bounds` 更新后才刷新
- ✅ 避免异步调用的时序问题
- ✅ `enableMarquee` 不会被意外重置

### 方案 2：保存 enableMarquee 状态

如果必须使用异步调用，可以保存状态：

```swift
func configUI(...) {
    self.resetAllStates()
    
    // 配置属性
    self.font = font
    self.gradientColors = colors
    
    // 保存状态
    let shouldEnableMarquee = (level > 0 || isAutoScroll)
    
    if shouldEnableMarquee {
        self.enableMarquee = true
        self.startAnimation()
        
        // 异步刷新
        DispatchQueue.main.async { [weak self] in
            guard let self = self else { return }
            
            // ✅ 关键：重新检查并设置 enableMarquee
            if !self.enableMarquee {
                print("⚠️ enableMarquee was reset, restoring...")
                self.enableMarquee = true
            }
            
            self.forceRefreshLayout()
        }
    }
}
```

### 方案 3：使用 restartMarqueeIfNeeded()

在 `willDisplay` 或其他合适的时机调用：

```swift
func tableView(_ tableView: UITableView, willDisplay cell: UITableViewCell, forRowAt indexPath: IndexPath) {
    if let cell = cell as? VIPCell {
        // ✅ 兜底：确保跑马灯启动
        cell.nameLabel.restartMarqueeIfNeeded()
    }
}
```

## 修复后的 GMGradientNameView

参考 `GMGradientNameView修复建议.swift` 文件，关键改动：

```swift
class GMGradientNameView: GMWaveGradientLabelView {
    private var needsRefreshLayout = false
    
    func configUI(font: UIFont, 
                  sameDirectionAnimationDuration: TimeInterval = 1.4,
                  level: Int, 
                  defaultColors: [UIColor], 
                  isAutoScroll: Bool = false) {
        
        // ✅ 1. 重置状态
        self.resetAllStates()
        
        // ✅ 2. 配置方向和对齐
        self.gradientDirection = GMLanguageChange.shared.isMiddleEast ? .horizontalLeftToRight : .horizontalRightToLeft
        self.textAlignment = GMLanguageChange.shared.isMiddleEast ? .right : .left
        
        // ✅ 3. 配置字体和颜色
        self.font = font
        self.gradientColors = getColors(level: level, defaultColors: defaultColors)
        
        // ✅ 4. 配置跑马灯
        if level > 0 || isAutoScroll {
            self.animationDuration = 4.5
            self.marqueeDirection = GMLanguageChange.shared.isMiddleEast ? .leftToRight : .rightToLeft
            self.sameDirectionAnimationDuration = sameDirectionAnimationDuration
            self.marqueeSpeed = 45
            self.marqueeDelay = 0.0
            self.marqueeGap = 10
            self.enableMarquee = true
            self.startAnimation()
            
            // ✅ 5. 标记需要刷新
            needsRefreshLayout = true
        }
    }
    
    override func layoutSubviews() {
        super.layoutSubviews()
        
        // ✅ 确保对齐方式正确
        self.textAlignment = GMLanguageChange.shared.isMiddleEast ? .right : .left
        
        // ✅ 6. 在布局完成后刷新
        if needsRefreshLayout {
            needsRefreshLayout = false
            forceRefreshLayout()
        }
    }
}
```

## 调试技巧

### 1. 检查 enableMarquee 变化

在 `WaveGradientLabel.swift` 中已经添加了调试输出：

```swift
var enableMarquee: Bool = false {
    didSet {
        #if DEBUG
        if enableMarquee != oldValue {
            print("⚠️ [enableMarquee] 值改变: \(oldValue) -> \(enableMarquee)")
            print("   调用栈:")
            Thread.callStackSymbols.prefix(5).forEach { print("      \($0)") }
        }
        #endif
        
        if enableMarquee {
            startMarquee()
        } else {
            stopMarquee()
        }
    }
}
```

这会打印出谁修改了 `enableMarquee`，帮助定位问题。

### 4. 常见问题诊断

#### 问题 1：enableMarquee 是 false

```
- enableMarquee: false
- marqueeDisplayLink: ❌ not running
```

原因：
- `enableMarquee` 被意外重置
- 没有调用 `enableMarquee = true`

解决：
- 检查调用栈，找到谁修改了 `enableMarquee`
- 确保 `configUI()` 中设置了 `enableMarquee = true`

#### 问题 2：文字不匹配

```
- text (property): "新文字"
- text (maskLabel): "旧文字"
- text match: ❌ MISMATCH!
```

原因：
- 没有调用 `resetAllStates()` 清理旧状态
- `text` 设置后没有触发 `rebuildAttributedTextAndLayout()`

解决：
- 在 `configUI()` 开始时调用 `resetAllStates()`
- 确保 `text` 在 `configUI()` 之前设置

#### 问题 3：文字太短

```
- textWidth: 100.0px
- containerWidth: 200.0px
- threshold: 1.0 (200.0px)
- needsScroll: false (❌)
```

原因：
- 文字宽度小于容器宽度，不需要滚动
- 这是正常行为，不是 bug

解决：
- 如果确实需要滚动，降低 `marqueeThreshold`：
  ```swift
  nameLabel.marqueeThreshold = 0.8  // 文字达到 80% 就滚动
  ```

#### 问题 4：bounds 不正确

```
- bounds: (0.0, 0.0, 0.0, 0.0)  // ❌ 宽度为 0
- textWidth: 150.0px
- containerWidth: 0.0px
```

原因：
- View 还没有布局
- Auto Layout 约束还没有应用

解决：
- 使用 `layoutSubviews` 中的 `needsRefreshLayout` 标志
- 或在 `willDisplay` 中调用 `restartMarqueeIfNeeded()`

## 总结

### 关键点

1. ✅ 使用 `needsRefreshLayout` 标志在 `layoutSubviews` 中刷新
2. ✅ 避免异步调用导致的时序问题
3. ✅ 在 `configUI()` 开始时调用 `resetAllStates()`
4. ✅ 先设置 `text`，再调用 `configUI()`
5. ✅ 使用 `debugMarqueeStatus()` 调试问题
