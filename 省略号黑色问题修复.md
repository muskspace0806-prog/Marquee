# 省略号黑色问题修复

## 问题描述

当文字超出控件宽度并显示省略号时，如果被截断的部分刚好包含表情符号，省略号会显示为黑色，而不是应用渐变效果。

例如：`1(★≧▽^)854🍒😍4` 被截断为 `1(★≧▽^)854...`，省略号是黑色的。

## 问题原因

我们的控件使用双层渲染：
- `maskLabel`：渲染普通文字（白色），作为渐变的 mask
- `emojiLabel`：渲染表情符号（彩色），叠加在上面

当文字被截断时：
1. `maskLabel` 使用 `.byTruncatingTail`，会显示省略号（白色，应用渐变）✅
2. `emojiLabel` 如果也显示省略号，会是黑色的（默认文字颜色），覆盖下面的渐变效果 ❌

## 解决方案

### 方案 1：emojiLabel 使用 .byClipping + 透明文字颜色（已实现）

```swift
// 初始化时设置
emojiLabel.lineBreakMode = .byClipping  // 不显示省略号
emojiLabel.textColor = .clear  // 默认透明，避免任何"意外"显示为黑色
```

这样：
- `maskLabel` 显示省略号（白色，应用渐变）✅
- `emojiLabel` 不显示省略号，即使显示也是透明的 ✅

### 方案 2：动态更新 lineBreakMode（已实现）

```swift
private func updateLineBreakMode() {
    // maskLabel：根据跑马灯状态设置
    let maskMode: NSLineBreakMode = enableMarquee ? .byClipping : .byTruncatingTail
    maskLabel.lineBreakMode = maskMode
    maskLabel2.lineBreakMode = maskMode
    
    // emojiLabel：始终使用 .byClipping
    // 原因：省略号应该只显示在 maskLabel 上（应用渐变效果）
    emojiLabel.lineBreakMode = .byClipping
    emojiLabel2.lineBreakMode = .byClipping
}
```

## 如果问题仍然存在

如果修改后省略号仍然是黑色，可能的原因：

### 原因 1：emojiLabel 的 z-index 问题

`emojiLabel` 叠加在 `maskLabel` 上面，即使设置了 `.byClipping`，它的边缘可能还是会显示。

**解决方案**：确保 `emojiLabel` 的 `textColor` 是透明的（已实现）。

### 原因 2：attributedText 的颜色覆盖

在 `rebuildAttributedTextAndLayout` 中，我们为表情符号设置了属性，但没有为省略号设置。

**解决方案**：在 `emojiAttr` 中为所有字符设置透明颜色。

```swift
// 在 emoji label 中显示原色（表情）或透明（文字）
emojiAttr.append(NSAttributedString(string: str, attributes: [
    .font: emojiFont,
    .foregroundColor: UIColor.clear  // ✅ 添加透明颜色
]))
```

### 原因 3：系统自动添加的省略号没有继承属性

UILabel 自动添加的省略号可能不会继承 `attributedText` 的颜色属性。

**解决方案**：使用自定义省略号。

```swift
// 方案 A：使用 NSAttributedString 的 truncation token
let paragraphStyle = NSMutableParagraphStyle()
paragraphStyle.lineBreakMode = .byTruncatingTail

let truncationToken = NSAttributedString(
    string: "...",
    attributes: [
        .font: font,
        .foregroundColor: UIColor.white  // 白色，应用渐变
    ]
)

// 但这需要手动计算截断位置，比较复杂
```

## 临时解决方案

如果问题仍然无法解决，可以使用以下临时方案：

### 方案 A：禁用表情符号时的省略号

```swift
// 如果文字包含表情符号，强制启用跑马灯
if text.contains(where: { characterContainsEmoji($0) }) {
    enableMarquee = true
}
```

### 方案 B：使用纯文字模式

```swift
// 禁用跑马灯时，不使用双层渲染
if !enableMarquee {
    // 只使用 maskLabel，不使用 emojiLabel
    emojiLabel.isHidden = true
    emojiLabel2.isHidden = true
}
```

但这会导致表情符号变成黑白的。

## 最终方案（如果上述都不行）

如果所有方案都不行，可能需要自定义省略号的渲染：

```swift
private func addCustomEllipsis() {
    // 1. 测量文字宽度
    let fullWidth = maskLabel.sizeThatFits(CGSize(width: .greatestFiniteMagnitude, height: bounds.height)).width
    
    if fullWidth > bounds.width {
        // 2. 计算可以显示多少字符
        var visibleText = ""
        var currentWidth: CGFloat = 0
        let ellipsisWidth = ("..." as NSString).size(withAttributes: [.font: font]).width
        
        for ch in text {
            let charWidth = (String(ch) as NSString).size(withAttributes: [.font: font]).width
            if currentWidth + charWidth + ellipsisWidth > bounds.width {
                break
            }
            visibleText.append(ch)
            currentWidth += charWidth
        }
        
        // 3. 手动添加省略号
        maskLabel.text = visibleText + "..."
        
        // 4. emojiLabel 只显示表情部分
        // ...
    }
}
```

但这个方案比较复杂，建议先尝试上面的简单方案。
