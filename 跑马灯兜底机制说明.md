# 跑马灯兜底机制说明

## 问题场景

在某些情况下，跑马灯可能因为时机问题没有正常启动：

1. **布局未完成**：设置 `enableMarquee = true` 时，view 的 bounds 还是 zero
2. **异步加载**：文字内容是异步获取的，设置时 view 还没有添加到 window
3. **复杂布局**：在 Auto Layout 或复杂视图层级中，布局时机不确定
4. **Cell 复用**：UITableView/UICollectionView 中 cell 复用导致状态混乱

## 兜底机制

### 1. 自动兜底（已内置）

WaveGradientLabelView 内置了多个兜底检查点：

#### a) `didMoveToWindow` 检查
```swift
override func didMoveToWindow() {
    super.didMoveToWindow()
    if window != nil {
        // 如果启用了跑马灯但没有启动，重新启动
        if enableMarquee && marqueeDisplayLink == nil {
            DispatchQueue.main.async { [weak self] in
                self?.startMarquee()
            }
        }
    }
}
```

#### b) `layoutSubviews` 检查
```swift
override func layoutSubviews() {
    super.layoutSubviews()
    
    if enableMarquee {
        // 如果启用了跑马灯但没有启动，重新启动
        if marqueeDisplayLink == nil && maskLabel.frame.width > bounds.width {
            startMarquee()
        }
    }
}
```

### 2. 手动兜底（推荐）

在关键时机手动调用兜底方法：

#### a) 在 `viewDidAppear` 中检查
```swift
override func viewDidAppear(_ animated: Bool) {
    super.viewDidAppear(animated)
    
    // 确保跑马灯正常启动
    marqueeLabel.restartMarqueeIfNeeded()
}
```

#### b) 在异步加载完成后检查
```swift
fetchText { [weak self] text in
    self?.label.text = text
    self?.label.enableMarquee = true
    
    // 延迟一帧，确保布局完成
    DispatchQueue.main.async {
        self?.label.restartMarqueeIfNeeded()
    }
}
```

#### c) 在 Cell 中使用
```swift
func configure(with text: String) {
    label.text = text
    label.enableMarquee = true
    
    // Cell 复用时，确保跑马灯重新启动
    label.restartMarqueeIfNeeded()
}

// 或在 willDisplay 中
func tableView(_ tableView: UITableView, willDisplay cell: UITableViewCell, forRowAt indexPath: IndexPath) {
    if let cell = cell as? MyCell {
        cell.label.restartMarqueeIfNeeded()
    }
}
```

## 使用示例

### 示例 1：基本使用（自动兜底）

```swift
let label = WaveGradientLabelView(frame: CGRect(x: 20, y: 100, width: 300, height: 60))
label.text = "这是一段很长的文字，会自动滚动"
label.enableMarquee = true
view.addSubview(label)

// ✅ 自动兜底：didMoveToWindow 和 layoutSubviews 会自动检查
```

### 示例 2：异步加载（手动兜底）

```swift
let label = WaveGradientLabelView(frame: CGRect(x: 20, y: 100, width: 300, height: 60))
view.addSubview(label)

// 异步获取文字
fetchLongText { text in
    label.text = text
    label.enableMarquee = true
    
    // ✅ 手动兜底：确保跑马灯启动
    DispatchQueue.main.async {
        label.restartMarqueeIfNeeded()
    }
}
```

### 示例 3：UITableViewCell（手动兜底）

```swift
class MyCell: UITableViewCell {
    let label = WaveGradientLabelView()
    
    func configure(with text: String) {
        label.text = text
        label.enableMarquee = true
        
        // ✅ 手动兜底：Cell 复用时重新启动
        label.restartMarqueeIfNeeded()
    }
}

// 在 ViewController 中
func tableView(_ tableView: UITableView, willDisplay cell: UITableViewCell, forRowAt indexPath: IndexPath) {
    if let myCell = cell as? MyCell {
        // ✅ 额外兜底：确保可见时跑马灯运行
        myCell.label.restartMarqueeIfNeeded()
    }
}
```

### 示例 4：Auto Layout（延迟兜底）

```swift
let label = WaveGradientLabelView()
label.text = "这是一段很长的文字"
label.enableMarquee = true
view.addSubview(label)

// Auto Layout 约束
label.translatesAutoresizingMaskIntoConstraints = false
NSLayoutConstraint.activate([
    label.leadingAnchor.constraint(equalTo: view.leadingAnchor, constant: 20),
    label.trailingAnchor.constraint(equalTo: view.trailingAnchor, constant: -20),
    label.topAnchor.constraint(equalTo: view.topAnchor, constant: 100),
    label.heightAnchor.constraint(equalToConstant: 60)
])

// ✅ 手动兜底：布局完成后检查
view.layoutIfNeeded()
label.restartMarqueeIfNeeded()
```

## API 说明

### `restartMarqueeIfNeeded()`

强制重新启动跑马灯（如果需要）。

**使用场景：**
- 异步加载文字后
- Cell 复用时
- 复杂布局完成后
- 任何怀疑跑马灯没有启动的场景

**特点：**
- 幂等操作：多次调用不会有副作用
- 智能检查：只在需要时才启动
- 安全调用：任何时候都可以调用

**示例：**
```swift
// 随时调用，不用担心重复启动
label.restartMarqueeIfNeeded()
```

## 最佳实践

1. **优先依赖自动兜底**：大多数情况下，内置的兜底机制已经足够
2. **关键时机手动兜底**：在 `viewDidAppear`、异步加载完成、Cell 复用时手动调用
3. **延迟一帧**：如果不确定布局是否完成，使用 `DispatchQueue.main.async`
4. **多次调用无害**：`restartMarqueeIfNeeded()` 可以放心多次调用
5. **调试时打印状态**：遇到问题时，打印相关状态信息

## 常见问题

### Q: 为什么跑马灯没有启动？

A: 可能原因：
1. 文字宽度 ≤ 容器宽度（不需要滚动）
2. bounds 还是 zero（布局未完成）
3. view 还没有添加到 window

解决方法：调用 `restartMarqueeIfNeeded()`

### Q: 跑马灯启动后又停止了？

A: 可能原因：
1. view 从 window 移除（自动停止）
2. `enableMarquee` 被设置为 false

解决方法：检查 `enableMarquee` 状态

### Q: Cell 复用时跑马灯混乱？

A: 在 `configure` 方法中：
```swift
func configure(with text: String) {
    label.text = text
    label.enableMarquee = true
    label.restartMarqueeIfNeeded()
}
```

## 总结

WaveGradientLabelView 提供了完善的兜底机制：
- ✅ 自动兜底：`didMoveToWindow` 和 `layoutSubviews`
- ✅ 手动兜底：`restartMarqueeIfNeeded()`
- ✅ 智能检查：只在需要时启动
- ✅ 安全可靠：多次调用无副作用

在大多数场景下，自动兜底已经足够。如果遇到特殊情况，在关键时机调用 `restartMarqueeIfNeeded()` 即可。
