# 跑马灯循环逻辑修复说明

## 问题描述

用户反馈两个问题：
1. 阿语情况下设置 `.leftToRight`，但依旧从右往左滚动
2. 文本过长滚动时，第二份文字刚出现，第一份就消失了（应该是连续的）

## 问题分析

### 问题 1：方向反了

**原因：**
- 初始位置设置错误：`.leftToRight` 时从 `x = -w` 开始
- 动画计算错误：`x1 = -w + offset`，导致向右移动但起点在容器外

### 问题 2：循环不连续

**原因：**
- `cycleLen = w + gap` 是一个完整循环的长度
- 但 `offset` 的范围是 `[0, cycleLen)`
- 当 `offset` 接近 `cycleLen` 时，第一份文字应该刚好移出，第二份文字刚好进入
- 之前的逻辑导致第二份文字的位置计算错误

**示例：**
```
文字宽度 w = 100
间距 gap = 20
循环长度 cycleLen = 120

期望的循环：
offset = 0:   label1 在 x=0,   label2 在 x=120
offset = 60:  label1 在 x=-60, label2 在 x=60
offset = 119: label1 在 x=-119, label2 在 x=1
offset = 0:   label1 在 x=0,   label2 在 x=120 (循环)

之前的错误：
offset = 0:   label1 在 containerWidth,   label2 在 containerWidth + 120
offset = 60:  label1 在 containerWidth-60, label2 在 containerWidth+60
// 当 label1 移出左边时，label2 还没进入右边！
```

## 修复方案

### 核心思路

**关键理解：**
- `.rightToLeft`：文字从屏幕右边外进入，向左移动，从左边离开
- `.leftToRight`：文字从屏幕左边外进入，向右移动，从右边离开

**实现要点：**
1. 初始位置必须在容器外（看不见）
2. 动画让文字移入容器，穿过容器，移出容器
3. 两份文字保持 `cycleLen` 的距离，确保连续

### 修复后的逻辑

#### 1. 初始位置（resetMarqueeToStartPosition）

```swift
switch marqueeDirection {
case .rightToLeft:
    // 从右往左：文字从容器右边外开始
    maskLabel.frame.origin.x = containerWidth  // 右边外
    maskLabel2.frame.origin.x = containerWidth + w + gap
    
case .leftToRight:
    // 从左往右：文字从容器左边外开始
    maskLabel.frame.origin.x = -w  // 左边外
    maskLabel2.frame.origin.x = -w - (w + gap)
}
```

#### 2. 动画更新（updateMarquee）

```swift
switch marqueeDirection {
case .rightToLeft:
    // 从右往左：从容器右边向左移动
    x1 = containerWidth - offset  // 向左移动（减少）
    x2 = x1 + cycleLen
    
case .leftToRight:
    // 从左往右：从容器左边向右移动
    x1 = -w + offset  // 向右移动（增加）
    x2 = x1 - cycleLen
}
```

## 关键点

### 1. 相对位置保持不变

无论 `offset` 如何变化，两份文字的相对位置始终是：
```swift
x2 = x1 + cycleLen  // rightToLeft
x2 = x1 - cycleLen  // leftToRight
```

### 2. 循环长度 = 文字宽度 + 间距

```swift
cycleLen = w + gap
```

这确保了：
- 当第一份文字完全移出时（移动了 `cycleLen`）
- 第二份文字刚好到达第一份的初始位置
- 此时 `offset` 重置为 0，循环重新开始

### 3. truncatingRemainder 确保连续

```swift
let offset = (CGFloat(elapsed) * speed).truncatingRemainder(dividingBy: cycleLen)
```

- `offset` 始终在 `[0, cycleLen)` 范围内
- 当 `offset` 达到 `cycleLen` 时，自动重置为 0
- 视觉上是连续的，没有跳跃


## 数学验证

### 从右往左（.rightToLeft）

```
初始：x1 = 0, x2 = cycleLen
移动：x1 = -offset, x2 = -offset + cycleLen

当 offset = 0:
  x1 = 0, x2 = cycleLen ✅

当 offset = cycleLen/2:
  x1 = -cycleLen/2, x2 = cycleLen/2 ✅

当 offset = cycleLen:
  x1 = -cycleLen, x2 = 0 ✅
  (此时 offset 重置为 0，x1 回到 0，x2 回到 cycleLen)
```

### 从左往右（.leftToRight）

```
初始：x1 = 0, x2 = -cycleLen
移动：x1 = offset, x2 = offset - cycleLen

当 offset = 0:
  x1 = 0, x2 = -cycleLen ✅

当 offset = cycleLen/2:
  x1 = cycleLen/2, x2 = -cycleLen/2 ✅

当 offset = cycleLen:
  x1 = cycleLen, x2 = 0 ✅
  (此时 offset 重置为 0，x1 回到 0，x2 回到 -cycleLen)
```

## 总结

修复后的行为：
- ✅ `.rightToLeft`：文字从右边进入，向左移动，循环连续
- ✅ `.leftToRight`：文字从左边进入，向右移动，循环连续
- ✅ 两份文字始终保持正确的相对位置
- ✅ 循环无缝衔接，没有跳跃或突兀
- ✅ 适用于所有语言（LTR 和 RTL）

修复的关键：
1. 统一坐标系，从 x=0 开始
2. 保持两份文字的相对位置不变（相差 cycleLen）
3. 使用 truncatingRemainder 确保循环连续
