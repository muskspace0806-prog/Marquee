# 省略号截取精度修复说明

## 问题描述
静态模式下，省略号截取不够准确，有时候文字明显超出宽度却没有显示省略号。

## 问题原因
1. **宽度计算不准确**：使用 `NSString.size(withAttributes:)` 计算字符宽度，对于表情符号和特殊字符不够准确
2. **简单累加宽度**：直接累加每个字符的宽度，没有考虑字符间距和排版因素
3. **缺少安全边距**：计算时没有预留足够的安全边距，导致边界情况下超出

## 修复方案

### 1. 使用 `boundingRect` 计算宽度
```swift
// ❌ 旧方案：不准确
let charWidth = (charStr as NSString).size(withAttributes: [.font: charFont]).width

// ✅ 新方案：更准确
let charAttr = NSAttributedString(string: charStr, attributes: [.font: charFont])
let charWidth = charAttr.boundingRect(
    with: CGSize(width: CGFloat.greatestFiniteMagnitude, height: bounds.height),
    options: [.usesLineFragmentOrigin, .usesFontLeading],
    context: nil
).width
```

### 2. 累积测试宽度
```swift
// ✅ 每次添加字符后测试总宽度
let testAttr = NSMutableAttributedString(attributedString: currentAttr)
testAttr.append(charAttr)
let testWidth = testAttr.boundingRect(
    with: CGSize(width: CGFloat.greatestFiniteMagnitude, height: bounds.height),
    options: [.usesLineFragmentOrigin, .usesFontLeading],
    context: nil
).width

if testWidth > maxWidth {
    break
}
```

### 3. 添加安全边距
```swift
// ✅ 添加 4 像素安全边距
let safetyMargin: CGFloat = 4
let maxWidth = bounds.width - ellipsisWidth - safetyMargin
```

## 修复效果
- ✅ 省略号截取更准确，不会超出控件宽度
- ✅ 正确处理表情符号和特殊字符
- ✅ 考虑字符间距和排版因素
- ✅ 边界情况下也能正确显示省略号

## 测试建议
测试以下场景：
1. 纯文字超出宽度
2. 包含表情符号的文字超出宽度
3. 包含特殊符号（★ ❤︎）的文字超出宽度
4. 混合文字、表情、符号的文字超出宽度
5. 边界情况（文字宽度刚好接近控件宽度）
