# 特殊符号和表情检测增强

## 问题描述

某些特殊符号（如 `❤︎`）显示为黑色，而不是彩色。

示例：`𝐉𝐚𝐍❤︎` 中的 `❤︎` 显示为黑色。

## 原因分析

### Unicode 变体选择器

`❤︎` 实际上是两个 Unicode 字符的组合：
- `❤` (U+2764) - 心形符号
- `︎` (U+FE0E) - 文本变体选择器

变体选择器用于指定符号的显示样式：
- `U+FE0E` - 文本样式（单色）
- `U+FE0F` - 表情样式（彩色）

### 检测问题

旧的检测逻辑：
```swift
private func isEmojiScalar(_ scalar: UnicodeScalar) -> Bool {
    if scalar.properties.isEmojiPresentation { return true }
    if scalar.properties.generalCategory == .otherSymbol, 
       scalar.properties.isEmoji { return true }
    if scalar.properties.isEmoji && (scalar.value >= 0x1F000) { return true }
    return false
}
```

问题：
1. `❤` (U+2764 = 0x2764) < 0x1F000，不满足第三个条件
2. 带有 `U+FE0E` 的 `❤︎` 可能不满足 `isEmojiPresentation`
3. 没有覆盖 `0x2600...0x26FF` 范围（杂项符号）

## 修复方案

### 增强的检测逻辑

```swift
private func isEmojiScalar(_ scalar: UnicodeScalar) -> Bool {
    // 1. 检查是否是表情展示形式
    if scalar.properties.isEmojiPresentation { return true }
    
    // 2. 检查是否是表情符号（包括文本样式的表情）
    if scalar.properties.isEmoji {
        // 排除变体选择器本身
        if scalar.value == 0xFE0E || scalar.value == 0xFE0F {
            return false
        }
        return true
    }
    
    // 3. 检查常见的符号范围
    switch scalar.value {
    case 0x2600...0x26FF,  // 杂项符号（包括 ❤ U+2764）
         0x2700...0x27BF,  // 装饰符号
         0x1F000...0x1FFFF: // 表情符号和象形文字
        return true
    default:
        return false
    }
}
```


### 忽略变体选择器

```swift
private func characterContainsEmoji(_ ch: Character) -> Bool {
    for scalar in ch.unicodeScalars {
        // 跳过变体选择器
        if scalar.value == 0xFE0E || scalar.value == 0xFE0F {
            continue
        }
        if isEmojiScalar(scalar) { return true }
    }
    return false
}
```

## 支持的符号范围

### 1. 杂项符号 (0x2600...0x26FF)

包括：
- ☀ (U+2600) - 太阳
- ☁ (U+2601) - 云
- ☂ (U+2602) - 雨伞
- ★ (U+2605) - 星星
- ♠ (U+2660) - 黑桃
- ♥ (U+2665) - 红心
- ♦ (U+2666) - 方块
- ♣ (U+2667) - 梅花
- ❤ (U+2764) - 心形

### 2. 装饰符号 (0x2700...0x27BF)

包括：
- ✂ (U+2702) - 剪刀
- ✈ (U+2708) - 飞机
- ✉ (U+2709) - 信封
- ✓ (U+2713) - 对勾
- ✗ (U+2717) - 叉号

### 3. 表情符号 (0x1F000...0x1FFFF)

包括所有标准表情符号：
- 😀 (U+1F600) - 笑脸
- 🐔 (U+1F414) - 鸡
- 🎨 (U+1F3A8) - 调色板
- 等等

## 关键改进

1. **扩展符号范围**：覆盖 `0x2600...0x26FF` 和 `0x2700...0x27BF`
2. **处理变体选择器**：正确识别带有 `U+FE0E` 或 `U+FE0F` 的符号
3. **统一处理**：所有表情和符号都使用系统字体，确保彩色显示

## 注意事项

### 文本样式 vs 表情样式

- `❤︎` (U+2764 + U+FE0E) - 文本样式，可能显示为单色
- `❤️` (U+2764 + U+FE0F) - 表情样式，显示为彩色

即使是文本样式，我们也将其识别为表情，使用系统字体渲染，这样可以：
- 保持符号的原始外观
- 避免被渐变覆盖
- 确保在不同设备上的一致性

### 数学字母符号

`𝐉𝐚𝐍` 这些是数学字母符号（Mathematical Alphanumeric Symbols），不是表情：
- 𝐉 (U+1D409) - 数学粗体大写 J
- 𝐚 (U+1D41A) - 数学粗体小写 a
- 𝐍 (U+1D40D) - 数学粗体大写 N

这些字符会被正常渲染为文字，应用渐变效果。
