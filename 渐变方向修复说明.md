# 渐变方向修复说明

## 问题描述

用户反馈：
1. `horizontalLeftToRight` 和 `horizontalRightToLeft` 的渲染方向与定义相反
2. `horizontalRightToLeft` 出现文本呼吸式收缩展开问题
3. 垂直和对角线方向无论选哪个，方向都保持旧的，没有任何改变

## 问题分析

### 核心问题

在 CAGradientLayer 中：
- `startPoint` 和 `endPoint` 定义了渐变色的分布方向（静态）
- 动画的移动方向决定了波浪的流动方向（动态）
- **如果 startPoint/endPoint 不同，会导致 gradientLayer 的 frame 需要不同的偏移，引起闪烁**

**关键理解：**
- 用户说的"左到右"是指**波浪的移动方向**，而不是渐变色的起始位置
- 波浪向右移动 = gradientLayer 向左移动（负方向）
- 波浪向左移动 = gradientLayer 向右移动（正方向）

### 修复方案

**统一原则：相同类型的方向使用相同的 points，只通过动画方向控制波浪流向**

```swift
// ✅ 水平方向：两个方向使用相同的 points
case .horizontalLeftToRight, .horizontalRightToLeft:
    return (CGPoint(x: 0, y: 0.5), CGPoint(x: 1, y: 0.5))

// ✅ 垂直方向：两个方向使用相同的 points
case .verticalTopToBottom, .verticalBottomToTop:
    return (CGPoint(x: 0.5, y: 0), CGPoint(x: 0.5, y: 1))

// ✅ 对角线（左上-右下 / 左下-右上）：使用相同的 points
case .diagonalDownRight, .diagonalUpRight:
    return (CGPoint(x: 0, y: 0), CGPoint(x: 1, y: 1))

// ✅ 对角线（右上-左下 / 右下-左上）：使用相同的 points
case .diagonalDownLeft, .diagonalUpLeft:
    return (CGPoint(x: 1, y: 0), CGPoint(x: 0, y: 1))
```

## 修复内容

### 1. 修改 `points` 计算属性

将相同类型的方向合并，使用相同的 points：

```swift
var points: (start: CGPoint, end: CGPoint) {
    switch self {
    case .horizontalLeftToRight, .horizontalRightToLeft:
        return (CGPoint(x: 0, y: 0.5), CGPoint(x: 1, y: 0.5))
        
    case .verticalTopToBottom, .verticalBottomToTop:
        return (CGPoint(x: 0.5, y: 0), CGPoint(x: 0.5, y: 1))
        
    case .diagonalDownRight, .diagonalUpRight:
        return (CGPoint(x: 0, y: 0), CGPoint(x: 1, y: 1))
        
    case .diagonalDownLeft, .diagonalUpLeft:
        return (CGPoint(x: 1, y: 0), CGPoint(x: 0, y: 1))
    }
}
```

### 2. 修改 `layoutGradientLayerForSeamlessTransform()`

每个方向单独设置 frame，确保动画从正确的位置开始：

```swift
switch gradientDirection {
case .horizontalLeftToRight:
    // 向右移动：从 x=-w 开始
    gradientLayer.frame = CGRect(x: -w, y: 0, width: w * 2, height: h)
    
case .horizontalRightToLeft:
    // 向左移动：从 x=0 开始
    gradientLayer.frame = CGRect(x: 0, y: 0, width: w * 2, height: h)

case .verticalTopToBottom:
    // 向下移动：从 y=-h 开始
    gradientLayer.frame = CGRect(x: 0, y: -h, width: w, height: h * 2)
    
case .verticalBottomToTop:
    // 向上移动：从 y=0 开始
    gradientLayer.frame = CGRect(x: 0, y: 0, width: w, height: h * 2)

case .diagonalDownRight:
    // 向右下移动：从 (-w, -h) 开始
    gradientLayer.frame = CGRect(x: -w, y: -h, width: w * 2, height: h * 2)
    
case .diagonalDownLeft:
    // 向左下移动：从 (0, -h) 开始
    gradientLayer.frame = CGRect(x: 0, y: -h, width: w * 2, height: h * 2)
    
case .diagonalUpRight:
    // 向右上移动：从 (-w, 0) 开始
    gradientLayer.frame = CGRect(x: -w, y: 0, width: w * 2, height: h * 2)
    
case .diagonalUpLeft:
    // 向左上移动：从 (0, 0) 开始
    gradientLayer.frame = CGRect(x: 0, y: 0, width: w * 2, height: h * 2)
}
```

### 3. 修改 `updateGradientFrame()` 动画方向

为每个方向单独设置动画移动方向（已反转）：

```swift
switch gradientDirection {
case .horizontalLeftToRight:
    dx = w * phase   // 向右移动，波浪向右
    dy = 0
    
case .horizontalRightToLeft:
    dx = -w * phase  // 向左移动，波浪向左
    dy = 0
    
case .verticalTopToBottom:
    dx = 0
    dy = h * phase   // 向下移动，波浪向下
    
case .verticalBottomToTop:
    dx = 0
    dy = -h * phase  // 向上移动，波浪向上
    
case .diagonalDownRight:
    dx = w * phase   // 向右下移动，波浪向右下
    dy = h * phase
    
case .diagonalDownLeft:
    dx = -w * phase  // 向左下移动，波浪向左下
    dy = h * phase
    
case .diagonalUpRight:
    dx = w * phase   // 向右上移动，波浪向右上
    dy = -h * phase
    
case .diagonalUpLeft:
    dx = -w * phase  // 向左上移动，波浪向左上
    dy = -h * phase
}
```

## 方向对照表（最终版本）

| 方向 | startPoint | endPoint | gradientLayer frame | 动画方向 (dx, dy) | 波浪流向 |
|------|-----------|----------|-------------------|-----------------|---------|
| horizontalLeftToRight | (0, 0.5) | (1, 0.5) | (-w, 0, 2w, h) | (+w·p, 0) | 向右 ✅ |
| horizontalRightToLeft | (0, 0.5) | (1, 0.5) | (0, 0, 2w, h) | (-w·p, 0) | 向左 ✅ |
| verticalTopToBottom | (0.5, 0) | (0.5, 1) | (0, -h, w, 2h) | (0, +h·p) | 向下 ✅ |
| verticalBottomToTop | (0.5, 0) | (0.5, 1) | (0, 0, w, 2h) | (0, -h·p) | 向上 ✅ |
| diagonalDownRight | (0, 0) | (1, 1) | (-w, -h, 2w, 2h) | (+w·p, +h·p) | 右下 ✅ |
| diagonalDownLeft | (1, 0) | (0, 1) | (0, -h, 2w, 2h) | (-w·p, +h·p) | 左下 ✅ |
| diagonalUpRight | (0, 0) | (1, 1) | (-w, 0, 2w, 2h) | (+w·p, -h·p) | 右上 ✅ |
| diagonalUpLeft | (1, 0) | (0, 1) | (0, 0, 2w, 2h) | (-w·p, -h·p) | 左上 ✅ |

**关键发现：**
- 向右移动的方向：gradientLayer.x 需要从 -w 开始
- 向下移动的方向：gradientLayer.y 需要从 -h 开始
- 这样可以确保动画从正确的位置开始，避免闪烁

## 技术细节

### 为什么相同类型的方向要使用相同的 points？

1. **避免闪烁**：不同的 points 可能需要不同的 frame 偏移，导致布局变化
2. **简化逻辑**：只需要改变动画方向，不需要改变 gradientLayer 的布局
3. **一致性**：相同类型的方向视觉效果更一致

### 动画移动方向与视觉效果的关系

| gradientLayer 移动 | 视觉上波浪流向 | 原因 |
|------------------|--------------|------|
| 向左（dx < 0）    | 向右流动      | 右边新内容进入可见区域 |
| 向右（dx > 0）    | 向左流动      | 左边新内容进入可见区域 |
| 向上（dy < 0）    | 向下流动      | 下边新内容进入可见区域 |
| 向下（dy > 0）    | 向上流动      | 上边新内容进入可见区域 |
