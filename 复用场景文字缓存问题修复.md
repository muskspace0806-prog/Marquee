# 复用场景文字缓存问题修复

## 问题现象

在 JXSegmentedView、UITableView、UICollectionView 等复用场景中：
- 打印的文字是旧的（如 "GG2434236"），不是当前设置的文字
- `textWidth` 是旧文字的宽度，不是新文字的宽度
- 导致跑马灯判断错误（应该滚动但没有滚动，或不应该滚动但启动了）

## 根本原因

### 原因 1：startMarquee() 使用了缓存的 frame.width

```swift
// ❌ 错误的做法
let textWidth = maskLabel.frame.width  // 这是旧的宽度！
```

问题：
- `resetAllStates()` 虽然清空了 `text`，但 `maskLabel.frame` 还保留着旧的宽度
- `updateTextSize()` 会重新计算，但 `startMarquee()` 在打印时使用的是旧的 `frame.width`

### 原因 2：updateTextSize() 的判断逻辑

```swift
private func updateTextSize() {
    if enableMarquee {
        let textSize = maskLabel.sizeThatFits(...)
        let needsScroll = textSize.width > bounds.width * marqueeThreshold
        
        if needsScroll {
            // 使用文字的实际宽度
            maskLabel.frame.size = textSize
        } else {
            // ✅ 不需要滚动：使用 bounds
            maskLabel.frame = bounds
        }
    }
}
```

问题：
- 如果 `enableMarquee = true` 但文字不需要滚动，`maskLabel.frame` 会被设置为 `bounds`
- 但 `startMarquee()` 在这之后被调用，使用的是 `maskLabel.frame.width`（已经是 bounds.width）
- 所以打印出来的 `textWidth` 是容器宽度，不是文字宽度

## 解决方案

### 修复 1：startMarquee() 使用实际测量的宽度

```swift
private func startMarquee() {
    updateTextSize()

    // ✅ 使用实际测量的文字宽度，而不是 maskLabel.frame.width
    let textWidth = maskLabel.sizeThatFits(CGSize(width: CGFloat.greatestFiniteMagnitude, height: bounds.height)).width
    let containerWidth = bounds.width
    let needsScroll = textWidth > containerWidth * marqueeThreshold
    
    // ... 后续逻辑
}
```

优点：
- 每次都重新测量文字宽度，不依赖缓存的 `frame.width`
- 确保使用的是当前文字的实际宽度

### 修复 2：增强 resetAllStates()

```swift
func resetAllStates() {
    // 1. 停止所有动画
    stopAnimation()
    stopMarquee()
    
    // 2. 重置属性
    text = ""
    enableMarquee = false
    
    // 3. ✅ 关键：清空 label 的内容，触发重新渲染
    maskLabel.text = nil
    maskLabel.attributedText = nil
    
    // 4. ✅ 关键：重置 frame 为 bounds，清理文字宽度缓存
    maskLabel.frame = bounds
    maskLabel2.frame = bounds
    emojiLabel.frame = bounds
    emojiLabel2.frame = bounds
    
    // 5. 强制重新布局
    setNeedsLayout()
    layoutIfNeeded()
}
```

优点：
- 彻底清理所有缓存
- 确保下次配置时使用的是全新的状态

## 常见问题

### Q1：为什么打印的文字还是旧的？

A：检查以下几点：
1. `resetAllStates()` 是否被调用？
2. `text` 是否在 `resetAllStates()` 之后设置？
3. `startMarquee()` 是否使用了 `maskLabel.sizeThatFits()` 而不是 `maskLabel.frame.width`？

### Q2：为什么 textWidth 是容器宽度而不是文字宽度？

A：可能的原因：
1. `updateTextSize()` 判断文字不需要滚动，设置了 `maskLabel.frame = bounds`
2. `startMarquee()` 使用了 `maskLabel.frame.width`（已经是 bounds.width）

解决方案：使用 `maskLabel.sizeThatFits()` 重新测量。

### Q3：如何确保每次都使用最新的文字？

A：
1. 在 `configUI()` 开始时调用 `resetAllStates()`
2. 在 `startMarquee()` 中使用 `sizeThatFits()` 重新测量
3. 不要依赖缓存的 `frame.width`

## 总结

### 关键修复

1. ✅ `startMarquee()` 使用 `sizeThatFits()` 而不是 `frame.width`
2. ✅ `resetAllStates()` 彻底清理所有缓存
3. ✅ `configUI()` 在开始时调用 `resetAllStates()`

### 修复后的效果

- 打印的文字是当前设置的文字，不是旧的缓存
- `textWidth` 是当前文字的实际宽度
- 跑马灯判断正确（该滚动就滚动，不该滚动就不滚动）
